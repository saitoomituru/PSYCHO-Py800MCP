# PSYCHO-Py800MCP
**AIに波形の眼を開眼させるMCPブリッジ**

> シビュラシステムは人の魂を測定した。  
> PSYCHO-Py800MCPは電気信号の魂を測定する。

Mac/Winメニューバー常駐のPythonアプリ。  
オシロスコープ・ロジアナ・WebCam・顕微鏡をMCPで統合し、  
AIがスケマティックを読んで波形の眼を持つ。  
引き金はエンジニアが引く。

---

## 設計哲学

### 責任の境界

```
Claude（指揮）
└── 係数計算・装備指示・次の測定点提案
　　「標的ネットTP3、ch1にプローブを当てて
　　　Startを押してください」← これしか言わない

エッジ端末（命令発行）
└── SCPIコマンド実行・実機ハードへの命令
　　← ここに責任が宿る

執行官/監視官（人間）
└── プローブを当てる・Startを押す
　　トリガーを引いた指が責任を持つ
```

**Claudeはコマンドを直接叩かない。エッジが叩く。**

### ゲシュタルト係数

LLMの倫理判断ではなく電気安全規格ベース（IEC 61508準拠）の定量リスク計算。

```
< 100   通常プローブ、即時執行可
100-299 差動プローブ推奨
300-399 差動プローブ必須＋高圧モード
400+    執行官2名体制、監視官承認必須
```

### 安全設計の思想

- **ナラティブ安全ではなく証拠保全型安全**
- 事故ゼロは不能という前提に立つ
- 事故った後まで設計する（証拠保全・責任ルート・リカバリー手順）
- 安全KPIは現場オペレーターが所有する
- Claudeの倫理観ではなくドメイン特化基準を使用
- ユーザーが判断基準を明示設定し責任を引き受ける

## アーキテクチャ

```
Claude（指揮）
　↓ 指示テキスト・係数
MCP橋渡し（PSYCHO-Py800MCP）
　↓ SCPIコマンド変換
エッジ端末（ZeroRoomLab / Hackintosh）
　↓ SCPI / sigrok
SIGLENT SDS1204X-E・実機ハード
　↓ 波形データ or「ハードあかん」
NumPy解析 → Claude次の指示
```

### ユーザーロール

| ロール | 役割 |
|--------|------|
| 監視官（デベロッパー） | Claude Codeでロジック・MCPからリモート操作 |
| 執行官（ハードエンジニア） | 実機にプローブ当て・Startを押す |

### 2つの動作モード

**フォワードモード** — KiCadスケマティック読み込み済み、係数が確定的  
**リバースモード** — 基板写真→Vision推定→信頼度スコア付き暫定スケマティック→反復収束
