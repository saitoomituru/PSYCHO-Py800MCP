# safety_design.md — 安全設計思想・責任構造

---

## 基本原則

**事故ゼロは不能という前提に立つ。**

だから「事故を防ぐ」ではなく「事故った後まで設計する」。

---

## 責任の構造

```
ツール製作者（齋藤みつる）
└── 「設定通りに動くツール」を作った責任

オペレーター/ユーザー
└── 「この基準で動かす」と設定した責任
　　判断基準を明示設定し、責任を引き受ける
　　= 育てた側の責任

Claude/Anthropic
└── 関係ない（安全判断層に入れない）
```

---

## なぜLLMベンダーの倫理観を使わないか

LLMベンダー（Anthropic/OpenAI）はソフトウェアサービスであり、  
物質世界（電圧・熱・慣性）に責任を持てない。  
だから「ナラティブ安全」（やらない理由を作る法務言語）しか語れない。

```
LLMベンダーの「安全」
└── 「危険なことはしません」← ビット列の話
　　「責任は負いかねます」← 賠償ゼロで逃げ切れる

現場の「安全」
└── 「感電死ゼロ」← 物理法則の話
　　人が死んだら遺族への賠償・刑事責任
```

この2つは次元が違う。ベンダーが現場の安全を語れる立場にない。

## PSYCHO-Py800MCPの安全設計

### 1. ゲシュタルト係数（定量リスク評価）

IEC 61508 SIL基準に準拠した動的リスク計算。  
「感覚」ではなく「数値」で判断基準を提示する。

詳細: `docs/interface_spec.md` 参照

### 2. 証拠保全（事故後の責任ルート確保）

- 全操作のタイムスタンプ付きSQLiteログ
- 執行許可ログ（誰が・いつ・何を・どのモードで）
- 異常時の波形データ自動保全
- ログは追記のみ・削除不可

これはドラレコ相当。事故後の責任ルートを確保する。

### 3. 装備指示（定量的装備選択）

係数に応じた装備指示を出す。  
「なんとなく」ではなく数値根拠のある装備選択。

```
係数 < 100  → 通常プローブ
係数 100+   → 差動プローブ推奨
係数 300+   → 差動プローブ必須＋高圧モード
係数 400+   → 執行官2名体制・監視官承認必須
```

### 4. リカバリー手順（蘇生・救急手配）

感電事故発生時のプロトコルをユーザーが設定する:

```json
{
    "emergency_contacts": ["119", "..."],
    "aed_location": "...",
    "recovery_protocol": "...",
    "auto_alert_threshold": 400
}
```

係数400+での操作開始時に緊急連絡先を画面表示。

## コントロール移譲の原則

```
リスク低  → AIが判断・自動実行可
リスク中  → AIが提案・人間が承認
リスク高  → AIは情報提供のみ・判断は完全人間
```

**リスクが高いほどユーザーに制御権を還元する。**

これはIEC 61508のSIL（Safety Integrity Level）と同一思想:

| SIL | 自動化レベル |
|-----|------------|
| SIL1 | 自動対応可 |
| SIL2 | 確認必要 |
| SIL3 | 人間判断必須 |
| SIL4 | 完全手動＋二重確認 |

## Claudeの限界の正直な定義

```
できること
├── データ解析・パターン照合
├── 係数計算・装備指示
├── 「標的ネットTP3、Startを押してください」
└── ログ保全・証拠確保

できないこと
├── 実機を見る・臭いを嗅ぐ・熱を感じる
├── 「なんか違う」を感じる
├── 波形が映らない原因を確定する
└── プローブを当てる
```

「ハードあかん」としか返ってこない時、Claudeは診断ツリーを提示するだけ。  
確認するのは現場の人間。

## ユーザー責任設定ファイル

`config/safety_profile.json` にユーザーが署名して保存。

```json
{
    "operator_name": "...",
    "operator_signature": "...",
    "accepted_at": "2026-06-21T08:32:10+09:00",
    "safety_standards": ["IEC61508", "JIS C 0508"],
    "custom_thresholds": {
        "gestalt_warning": 200,
        "gestalt_stop": 350
    },
    "emergency_contacts": ["119"],
    "aed_location": "ZeroRoomLab 玄関横",
    "recovery_protocol": "AEDまで30秒・119番同時発報"
}
```

この設定ファイルが存在しない場合、ツールは起動しない。
