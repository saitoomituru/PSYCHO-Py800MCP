## 基本原則

現行方針では、安全を責任の押し付け先ではなく、事故前に人員、装備、時間、停止・復旧能力を構成する
engineering capabilityとして扱う。防げる条件は測定前に塞ぎ、残余riskとunknownを表示し、実行後の
recoveryまで設計する。現行正本は
[`architecture/prospective-meta-safety.md`](architecture/prospective-meta-safety.md)と
[`architecture/measurement-plan-approval.md`](architecture/measurement-plan-approval.md)である。

以下の「責任」中心記述は2026-06-21時点の初期思想記録で、現行の実装入力へ使用しない。

## なぜLLMベンダーの倫理観を使わないか

LLMベンダー（Anthropic/OpenAI）はソフトウェアサービスであり、  
物質世界（電圧・熱・慣性）に責任を持てない。  
だから「ナラティブ安全」（やらない理由を作る法務言語）しか語れない。

```
LLMベンダーの「安全」
└── 「危険なことはしません」← ビット列の話
　　「責任は負いかねます」← 賠償ゼロで逃げ切れる

現場の「安全」
└── 「感電死ゼロ」← 物理法則の話
　　人が死んだら遺族への賠償・刑事責任
```

この2つは次元が違う。ベンダーが現場の安全を語れる立場にない。

## PSYCHO-Py800MCPの安全設計

### 1. ゲシュタルト係数（旧: 定量リスク評価）

初期案では動的リスク計算としていたが、現行方針では非専門者向けの順序メタ表記へ変更した。
現時点でIEC 61508/JIS C 0508への適合または認証取得を主張しない。
係数は円、確率、事故率、損害額、ROI、企業価値へ換算できず、技術ゲートにも使用しない。

詳細: `docs/interface_spec.md` 参照

### 2. 証拠保全（事故後の責任ルート確保）

- 全操作のタイムスタンプ付きSQLiteログ
- 執行許可ログ（誰が・いつ・何を・どのモードで）
- 異常時の波形データ自動保全
- ログは追記のみ・削除不可

これはドラレコ相当。事故後の責任ルートを確保する。

### 3. 装備指示（旧: 定量的装備選択）

以下の係数表は初期思想の履歴であり、現行の実行権限ではない。係数の大小にかかわらず、AIによる
測定開始と垂直レンジ／入力経路変更にはDev CheckとEngineer Checkの両方を要求する。時間軸だけの
変更はデータ品質操作として分離する。

係数GUIの主目的は、技術非専門の管理者、人事、出資者へ現場modeと必要人員を短時間で伝えること。
係数は技術ゲートではなく、技術ゲートの状態を表示するメタ情報である。

係数に応じた装備指示を出す。  
「なんとなく」ではなく数値根拠のある装備選択。

```
係数 < 100  → 旧案: 通常プローブ（自動実行許可を意味しない）
係数 100+   → 差動プローブ推奨
係数 300+   → 差動プローブ必須＋高圧モード
係数 400+   → 執行官2名体制・監視官承認必須
```

### 4. リカバリー手順（蘇生・救急手配）

感電事故発生時のプロトコルをユーザーが設定する:

```json
{
    "emergency_contacts": ["119", "..."],
    "aed_location": "...",
    "recovery_protocol": "...",
    "auto_alert_threshold": 400
}
```

係数400+での操作開始時に緊急連絡先を画面表示。

## コントロール移譲の原則

```
保存済み解析・観測読取      → AIが反復実行可
data-quality変更            → 記録付きで実行候補
破壊方向・影響不明の操作    → Dev Check＋Engineer Check＋plan承認
```

**リスクが高いほどユーザーに制御権を還元する。**

以下は初期案として置かれたプロジェクト内の制御移譲レベルであり、IEC 61508のSIL定義ではない。
SILとの直接対応表として使用しない。

| 旧ローカルレベル | 自動化レベル案 |
|-----|------------|
| L1 | 自動対応候補 |
| L2 | 確認必要 |
| L3 | 人間判断必須 |
| L4 | 完全手動＋二重確認 |

## Claudeの限界の正直な定義

```
できること
├── データ解析・パターン照合
├── 係数計算・装備指示
├── 「標的ネットTP3、Startを押してください」
└── ログ保全・証拠確保

できないこと
├── 実機を見る・臭いを嗅ぐ・熱を感じる
├── 「なんか違う」を感じる
├── 波形が映らない原因を確定する
└── プローブを当てる
```

「ハードあかん」としか返ってこない時、Claudeは診断ツリーを提示するだけ。  
確認するのは現場の人間。
